

# **Arden Engine — Concept Overview (High-Level Draft)**

*(гибридный воксельно-векторный движок, R&D-архитектура)*

Arden Engine — экспериментальный open-source движок, который сочетает **воксели**, **векторные структуры**, **иерархическую топологию** и **динамические контейнеры** в единой математической модели.
Цель — создать ядро, которое одинаково хорошо работает с:

* разрушаемыми мирами,
* процедурной генерацией,
* динамическими объектами,
* гибридом «воксель + меш»,
* оптимизациями под SVO/LOD.

---

# **1. Иерархическая топология мира**

Мир разбит на фиксированное дерево контейнеров:

```
Region → Block → Chunk → Octochunk → Voxel
```

Каждый уровень:

* имеет свои строгие размеры (страйды),
* полностью обратим (биекция),
* подходит под flat-indexing и Morton-порядок,
* идеально ложится на ECS + GPU-модели данных.

Внутренние координаты — **только положительные**; знак кодируется через **Octant (A–I)**, что упрощает математику и делает адресацию глобально обратимой.

---

# **2. Routing Layer (адресация)**

Route = путь точки в мире:

```
Region / Octant / Block / Chunk / Octo / v(x|z|y)
```

Его функция:

* унифицировать адреса,
* быть ключом для всего мира (генерация, физика, синхронизация, оптимизации),
* давать O(1) переход: Route → XYZ и обратно,
* позволять локальные операции: сравнение, вложенность, смещение, up/down-переходы.

Это «протокол адресов» всего движка.

---

# **3. Rotation / Orientation Layer**

Локальная ротация контейнеров и объектов:

* реализована как преобразование осей в адресной системе,
* не ломает топологию,
* работает в дискретных поворотах: 0°, 90°, 180°, 270°,
* важна для:

  * модульных структур,
  * динамических объектов,
  * копирования/вставки областей,
  * поворота динамических контейнеров (DYN).

---

# **4. DYN-Containers (динамическая геометрия)**

Это переносные «локальные миры» внутри основного мира.

Пример: предметы, техника, двери, механизмы, NPC, модули зданий.

DYN-контейнер:

* имеет собственную мини-топологию и локальную воксельную сетку,
* легко переносится и вращается,
* хранит своё содержимое как атомарный блок,
* может быть выгружен в архив (sleep),
* взаимодействует с миром через ААBB/OBB + Route.

Именно на DYN строится вся «динамика» гибридного движка.

---

# **5. Two-Module Hybrid Core: Voxel + Vector**

Движок использует 2 параллельных модуля:

## **5.1. Voxel Module**

* хранение,
* разрушение,
* SVO/LOD,
* генерация,
* физика объёмов,
* быстрая маршрутизация по топологии.

## **5.2. Vector/Mesh Module**

* триангуляция,
* гибридный рендер,
* упрощение мешей,
* компоновка динамических моделей.

Оба модуля используют **один язык адресации (Route/Topology)** — поэтому вектор и воксель не конфликтуют.

---

# **6. EQ-Kernel (Equilibrium Kernel)**

*«Мат-слой» — сердце движка.*

Задача EQ:

* держать **целостность мира**,
* отслеживать активность,
* координировать модули (воксель, вектор),
* готовить данные для рендера,
* управлять динамическими контейнерами,
* быть основой симуляции.

EQ не рендерит — он *живет сам по себе*, без графики.

---

# **7. HAOS — Hybrid Adaptive Optimization System**

Не «чемодан трюков», а адаптивный слой между EQ и GPU/шиной:

* LOD,
* активность регионов (sleep/wake),
* климат зон интереса,
* приоритеты рендера,
* упрощение данных,
* динамическая выборка чанков,
* BVH/SVO интеграции.

HAOS ≠ ECS.
Он работает над ECS как **динамический балансировщик нагрузки**.

---

# **8. BUS-Layer (шина)**

Буфер обмена между ядром (EQ+HAOS) и GPU:

* собирает пачки данных,
* группирует чанки и DYN-контейнеры,
* сортирует очереди по приоритетам,
* подготавливает батчи для WGPU/Vulkan.

---

# **9. Render-Pipeline (WGPU)**

Использует:

* меши от векторного модуля,
* триангуляцию вокселей,
* SSAO / PFO / шейдерные фильтры,
* GPU-pipeline под батчинг чанков.

PFO — фильтры постобработки, стиль, освещение.

---

# **10. Плейграунд и MVP**

Первый шаг к запуску движка:

* песочница с навигацией,
* инструменты генерации,
* debug-панели,
* инспекция Route/Chunk/SVO,
* тесты DYN-контейнеров и ротации,
* небольшая 2.5D-демо-игра.

---

# **Зачем всё это?**

Arden Engine предназначен для:

* игр с разрушаемым миром,
* гибридной геометрии (воксель + меш),
* симуляторов,
* исследовательских проектов,
* R&D экспериментов,
* генерации миров,
* прототипирования сложных механик.

Главная особенность — **единая математическая модель**, которая даёт:

* абсолютную целостность данных,
* предсказуемость,
* согласованность между системами,
* отсутствие плавающих ошибок,
* высокую масштабируемость.
